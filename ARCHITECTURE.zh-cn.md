# 架构设计

OpenClaw Recall 的内部设计、组件概览与记忆质量过滤机制。

---

## 记忆质量过滤机制

OpenClaw Recall 将记忆质量作为运行时的一等关注点，过滤在写入阶段和召回阶段双重生效。

### 写入时过滤（以下内容不会进入持久记忆）
- 发送者元数据、定时任务/心跳记录、控制平面标签
- Wrapper 文本、调试注释、脚手架片段
- 纯情绪类低价值内容

### 召回时抑制（以下内容不会主导召回结果）
- 旧的噪声行覆盖当前有效记忆
- 过期或已被覆盖的行挤占有效记忆位置
- 内部 wrapper / 调试文本泄漏进正常回复

### 稳定偏好提取倾向
`偏直接` · `偏执行导向` · `偏中文` · `偏简洁` · 结构化报告偏好

### 冲突处理支持
- 稳定偏好的覆盖更新
- 常见事实的更新
- 减少过时行对长期召回的污染

---

## 召回设计

混合召回使用 RRF 融合策略，平衡三种信号来源：

1. **稳定偏好** — 跨 session 持久存活的用户长期偏好
2. **项目上下文** — 当前项目状态与任务定义
3. **活跃任务/会话上下文** — 当前正在发生的事情

附加机制：
- **候选池扩展** — 重排前扩大初始候选集
- **MMR 多样化** — 减少重复偏好主导的召回结果
- **召回门控** — 对指令型 prompt 完全跳过记忆检索，避免无意义的计算
- **关系感知拼接** — 在 import 或 restore 后重建项目/任务记忆的关联关系

---

## 压缩设计

工具输出压缩保留有意义的结构：
- 命令及其输出
- 错误堆栈
- 代码块
- 半结构化段落（键值对、表格、标题）

压缩前自动解包 Provider 风格的 wrapper 层，确保压缩结果反映的是实际内容而非 JSON 壳子。

---

## 导入设计

长文导入将超长记忆和对话段落分块处理，使信号分布在多个可召回单元中，而不是堆在一个大 blob 里。

导入质量控制：
- `rejectedNoise` — 被过滤为噪声的内容
- `rejectedSensitive` — 被过滤为敏感内容
- `uncertainCandidates` — 标记为需人工审查的内容

普通导入不会把语义记忆静默晋升为 `shared` 范围。导出的插件数据在重新导入时保留原始 scope 元数据。

---

## 记忆类型

| 类型 | TTL | 用途 |
|---|---|---|
| `preference` | 较长 | 稳定用户偏好（名称、语言、风格） |
| `semantic` | 中等 | 事实性和领域知识 |
| `episodic` | 较短 | 近期 session 中发生的事情 |
| `session_state` | 当前 session | 活跃任务与上下文状态 |

---

## Prompt 预算

- 默认上下文预算：`2400` tokens
- 近期轮次窗口：`6` 轮
- 超过轮次阈值后自动触发历史摘要
- 预算强制执行，Recall 不会超出配置的预算上限

---

## 数据精度来源

- `promptTokensSource=exact` — Provider 提供用量元数据时
- `promptTokensSource=estimated` — Provider 未提供时
- `compressionSavingsSource=estimated` — 启发式对比
- `toolTokensSavedSource=estimated` — 启发式对比
